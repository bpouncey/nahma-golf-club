## 0. Optimization Cadence and Re-Verification Risk

### The Name Discrepancy: What to Know First

The current GBP business name is **Nahma Golf Course**. The website domain, email address, and UPGA listing all use **Nahma Golf Club**. This inconsistency is a real NAP problem and should be resolved, but the name field is the single highest-risk edit on the entire profile. Changing it is the one action most likely to trigger a re-verification requirement.

**The recommendation:** Do not change the name first. Build out everything else in the profile over the first two to three weeks, establish that the profile is active and healthy, and then make the name change as a standalone edit with nothing else touched that day. If re-verification is triggered, a well-built profile is far easier to recover than a bare one.

**On the "Course" vs. "Club" question specifically:** Google does not automatically flag this as a spam edit the way it would a keyword-stuffed name change (e.g., "Nahma Golf Course Rapid River MI Tee Times"). It is a legitimate business name correction. The risk is moderate, not high. But because the profile may not have been actively managed recently, the first edits back will draw more algorithmic scrutiny than edits on a regularly maintained profile. Ease in first.

---

### What Triggers Re-Verification

Understanding which edits carry risk lets you sequence work intelligently.

**High risk (can trigger re-verification):**
- Business name changes
- Physical address changes
- Primary phone number changes
- Primary category changes (occasionally)
- A sudden burst of core field edits on a dormant or recently transferred profile

**Low to no risk (safe to do immediately and in bulk):**
- Business description
- Photos
- Services
- Products
- Attributes
- Q&A posts
- GBP posts (What's New, Event, Offer)
- Secondary categories
- Hours updates (minor risk only if dramatically different from what's verified)
- Secondary phone or website URL (low risk)

**Agency account note:** You are making these edits from an agency-managed account rather than the verified owner account. This does not change re-verification risk in itself, but it does mean that if re-verification is triggered, the process will require action from whoever holds the verified owner access. Confirm before starting that you have a clear path to the owner if that happens. If the owner account has not logged in for an extended period, nudge them to log in and confirm access before you begin any high-risk edits.

### Recommended Rollout Cadence

Work through these phases in order. Do not jump to Phase 3 until Phase 2 has been live for at least a week with no flags or issues.

**Phase 1: Zero-Risk Foundation (Day 1)**
These can all be done in a single session. None of these touch core identity fields.

- Upload cover photo (aerial drone image)
- Upload 5-10 additional course and clubhouse photos with descriptive filenames
- Add the updated business description
- Add all attributes from Section 4
- Add secondary categories (Restaurant, Golf club)

**Phase 2: Services, Products, and Q&As (Days 3-7)**
Space these across a few days rather than doing everything at once. There is no technical reason you cannot do them in one session, but gradual activity on a previously inactive profile looks more natural to Google's quality systems.

- Add all services from Section 3 (spread across 2-3 sessions if comfortable)
- Add all products from Section 3a
- Seed the Q&A list from Section 5 (5-7 per session works fine)
- Publish your first GBP post

**Phase 3: Name Change (Week 2-3, Standalone)**
Make this edit on its own. Do not bundle it with any other changes that day.

- Change business name from "Nahma Golf Course" to "Nahma Golf Club"
- Monitor the profile for 48-72 hours afterward
- If a re-verification prompt appears, follow the postcard or video verification process promptly

After the name change is confirmed and stable, update all external citations to match (see Section 9 for the full audit list). Do the citation cleanup after the GBP name is locked in, not before, so the GBP becomes the confirmed source of truth that other directories align to.

**Phase 4: Schema and Website (Ongoing, Parallel)**
These are website-side tasks that do not touch the GBP directly and carry no re-verification risk.

- Deploy GolfCourse JSON-LD schema on the homepage
- Add Event schema for each 2026 calendar event
- Ensure NAP on the website footer matches GBP exactly once the name change is live

**Phase 5: Citation Cleanup (After Phase 3 is Confirmed)**
- Work through the citation audit in Section 9
- Prioritize Apple Maps (city mismatch), Yelp, GolfLink, and Facebook first
- Add missing directories (Foursquare, TripAdvisor, Nextdoor)
